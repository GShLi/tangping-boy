# Redis

## 问题

1. 使用 Redis 和 CaffeineCache 的区别是什么？

    Caffeine 和 Redis 作为两种不同类型的缓存解决方案，它们之间存在一些明显的区别：

    缓存存储位置：
        Caffeine 是一个纯 Java 缓存库，缓存数据存储在 JVM 内存中。
        Redis 是一个独立的缓存服务器，缓存数据存储在独立的 Redis 服务器内存中。

    持久化：
        Caffeine 缓存数据存在于 JVM 内存中，重启应用程序会导致缓存数据丢失。
        Redis 支持数据持久化，可以将缓存数据定期保存到硬盘上，即使 Redis 服务重启，缓存数据也不会丢失。

    分布式支持：
        Caffeine 是一个单机缓存解决方案，无法实现分布式部署。
        Redis 是一个分布式缓存系统，可以通过主从复制和哨兵机制实现高可用和扩展。

    性能：
        Caffeine 基于最先进的缓存算法，如 LRU 和 TinyLFU, 具有非常高的性能。
        Redis 作为一个内存数据库，也提供了非常高的读写性能。

    功能：
        Caffeine 主要提供基础的缓存功能，如自动加载、淘汰策略等。
        Redis 除了提供缓存功能外，还提供了丰富的数据结构和功能，如发布订阅、流处理等。
