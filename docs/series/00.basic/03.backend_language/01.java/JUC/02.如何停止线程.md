# 如何停止线程

## 停止线程的方式

Java 本身并没有提供优雅停止线程的 API

为什么 Java 没有提供强制停止线程的 API 呢？

因为在停止线程时我们并不知道线程此时处于什么状态，有可能是在写文件，也有可能是在传输数据，此时强制停止线程让任务中断可能会产生一些安全问题：比如文件写入失败，数据丢失等。

## 那该如何停止线程呢？

方式一：使用 interrupt 告知线程需要停止。

方式二：通过 volatile 标记位告知线程需要停止。

```java
public class InterruptRunnable implements Runnable {

  @Override
  public void run () {
    int num = 10000;
    for (int i = 0; i < num; i++) {
      if (i % 500 == 0) {
        System.out.println ("InterruptRunnable 线程：----" + i);
      }
    }
  }

}

public class VolatileRunnable implements Runnable {

  /**
   * 用于标记是否需要停止线程
   */
  public volatile boolean canceled = false;

  @Override
  public void run () {
    int num = 10000;
    for (int i = 0; i < num && !canceled; i++) {
      if (i % 500 == 0) {
        System.out.println ("VolatileRunnable 线程：----" + i);
      }
    }
  }
}

public class Demo02Application {

  public static void main (String [] args) throws InterruptedException {

    VolatileRunnable volatileRunnable = new VolatileRunnable ();
    Thread volatileThread = new Thread (volatileRunnable);
    volatileThread.start ();
    Thread.sleep (100);
    volatileRunnable.canceled = true;

    InterruptRunnable interruptRunnable = new InterruptRunnable ();
    Thread interruptThread = new Thread (interruptRunnable);
    interruptThread.start ();
    Thread.sleep (100);
    interruptThread.interrupt ();
  }

}
```

## 为什么用 volatile 标记位的停止方式是错误的？

```java
public class VolatileRunnable implements Runnable {

  /**
   * 用于标记是否需要停止线程
   */
  public volatile boolean canceled = false;

  @Override
  public void run () {
    int num = 10000;
    for (int i = 0; i < num && !canceled; i++) {
      if (i % 500 == 0) {
        System.out.println ("VolatileRunnable 线程：----" + i);
      }
    }
  }
}
```

通过 volatile 标记位实现停止线程的原理是不断的去读取标记为的状态来判断当前线程执行的任务要不要停止。

当任务执行过程中无法读取到 volatile 标记位的状态时，线程就无法停止。

比如将上述代码进行改造，该任务每次打印输出后都会 sleep 100 秒，在这 100 秒中线程是不会执行到读取 volatile 标记位状态的，进而无法立即停止线程，只能等待线程苏醒。

```java
public class VolatileRunnable implements Runnable {

  /**
   * 用于标记是否需要停止线程
   */
  public volatile boolean canceled = false;

  @Override
  public void run () {
    int num = 10000;
    for (int i = 0; i < num && !canceled; i++) {
      if (i % 500 == 0) {
        System.out.println ("VolatileRunnable 线程：----" + i);
        Thread.sleep (100000);
      }
    }
  }
}
```

那 sleep 时能否感知到 interrupt 中断呢？

答案是可以的，当我们调用 interrupt 方法时，sleep 会抛出

```java
public class InterruptRunnable implements Runnable {
  @Override
  public void run () {
    try {
      int num = 10000;
      for (int i = 0; i < num; i++) {
        if (i % 500 == 0) {
          System.out.println ("InterruptRunnable 线程：----" + i);
          Thread.sleep (10000);
        }
      }
    } catch (InterruptedException e) {
      System.out.println ("InterruptRunnable 线程停止！");
    }
  }
}
```

我们可以通过加 try-catch 来捕获异常并优雅的结束线程。
