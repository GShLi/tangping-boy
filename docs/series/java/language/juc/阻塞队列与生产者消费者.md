# 阻塞队列与生产者消费者

## 生产者消费者模式

生产者生产的速度和消费者消费的速度不一定是匹配的，因此需要在生产者和消费者之间构建一个缓冲区。

这个缓冲区一般用阻塞队列实现。生产者将生产的数据放入队列，消费者去消费队列中的数据。

当阻塞队列满了之后生产者就进入阻塞状态。队列非空就 notify 消费者。

同理，阻塞队列空了之后消费者就进入阻塞状态。队列空了就 notify 生产者。

因此需要两把锁，productLock 和 consumerLock。

队列空时，consumerLock 调用 wait，productLock 调用 notify。

队列满时，productLock 调用 wait，consumerLock 调用 notify。
