# 如何创建线程

## 创建线程的唯一方式

创建线程的唯一方式是构建一个 Thread 类

## 如何启动一个线程

Thread 中有两个方法：`run ()` 和 `start ()`

这两个方法有什么区别呢？

当我们调用 start 方法时，Thread 会启动线程并执行 run 方法中的代码，新线程会和主线程同步执行，互不影响。

而调用 run 方法时，则只执行 run 方法中的逻辑，主线程会等待 run 方法执行完毕后再继续向后执行。

## 构建 Thread 的两种基本方式

1. 继承 Thread 类并重写 run 方法

    ```java
    public class CustomThread extends Thread {

    @Override
    public void run () {
        System.out.println ("重写了 Thread 的 run 方法！");
    }
    }

    public class Demo01Application {

    public static void main (String [] args) {
        CustomThread customThread = new CustomThread ();
        customThread.start ();
    }

    }
    ```

2. 创建 Runnable 的实现类并传递给 Thread

    ```java
    public class CustomRunnable implements Runnable {

    @Override
    public void run () {
        System.out.println ("实现了 Runnable 接口，并在 run 方法中写线程执行的逻辑！");
    }
    }

    public class Demo01Application {

    public static void main (String [] args) {
        Thread runnableThread = new Thread (new CustomRunnable ());
        runnableThread.start ();
    }

    }
    ```

    通过查看 Thread 类的源代码我们可以发现，run 方法相当于代理了 targe 对象的 run 方法，而 target 则是 `Runnable` 类型的。

    Thread 其中一个构造函数的参数就是 Runnable 类型的 target，上方的第二种构建 Thread 类的方式就是调用了改构造函数。

    ```java

    /* What will be run. */
    private Runnable target;

    public Thread (Runnable target) {
        init (null, target, "Thread-" + nextThreadNum (), 0);
    }

    @Override
    public void run () {
        if (target != null) {
            target.run ();
        }
    }
    ```

这两种构建 Thread 类的方式有什么区别和优缺点？

通过代码我们可以看出，第一种方式 (继承并重写 Thread 类中的 run 方法) 将线程执行的任务和线程本身进行强绑定，改线程只可以执行固定的任务，当任务执行完毕后线程只能销毁。若要执行其他任务则需要新建线程，可能会增大线程管理的开销。

第二种方式则对二者进行了解耦，将线程和任务分离开来，权责分明。
