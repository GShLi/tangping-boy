# 牛刀小试:我的第一个容器化应用

## 部署一个 Nginx 应用

### 定义一个应用

Kubernetes 建议我们将容器的定义、参数、配置等信息存储在一个 YAML 文件中，并通过 `kubectl create -f <fileName>`运行。

案例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 2
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.7.9
          ports:
            - containerPort: 80
```

像上面这样一个 YAML 文件在 K8S 中就是一个 API 对象，我们给该对象的各个字段填好值交给 K8S 之后，K8S 会创建响应的 API 资源

字段解释：

1. kind： 资源类型

Kubernetes API 对象由两个部分构成:Metadata 和 Spec，前者存放的是该对象的元数据，一般为 API 对象的通用属性。后者 Spec 为该对象特有的属性，用于描述其所表达的功能。

### 获取 API 对象

我们可以通过 `kubectl get` 命令从 Kubernetes 获取 API 对象

`-l` 参数可以指定选择器(Selector)

例如 `kubectl get pod -l app=nginx` 用于获取所有 app=nginx 的 Pod 对象

获取结果:

```text
NAME                                READY   STATUS    RESTARTS   AGE
nginx-deployment-5754944d6c-97scq   1/1     Running   7          46m
nginx-deployment-5754944d6c-tqgbg   1/1     Running   7          46m
```

从结果中我们可以看到有两个正在运行中的 Pod（nginx-deployment-5754944d6c-97scq 和 nginx-deployment-5754944d6c-tqgbg）

### 问题

1. Pod 创建后一直处于 Pending 状态

   环境信息：只有一个 Master 节点

   产生原因：K8S 默认不允许在 Master 节点部署容器

   解决方案：`kubectl taint nodes --all node-role.kubernetes.io/master-`

## Volume 初体验

```yaml {20-25}
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 2
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.8
          ports:
            - containerPort: 80
          volumnMounts:
            - mountPath: /usr/share/nginx/html
              name: nginx-vol
      volumns:
        - name: nginx-vol
          emptyDir: {}
```

上述文件中添加一个与 containers 同级的 volumes 配置，该配置创建了一个名为 nginx-vol 类型为 emptyDir 的 volume

emptyDir 类型的 volume: Kubernetes 会创建一个临时目录并挂载到容器中，使用该类型的主要原因时 Kubernetes 不想依赖 Docker 自己创建的那个 `_data` 目录

volumeMounts 则配置了该容器要挂载哪些目录

mountPath 指定了挂载到容器中的路径，name 指定了将要挂载到容器中的宿主机的目录，在此我们指定了下方定义的名为 nginx-vol 的 volume

### 挂载宿主机目录

Kubernetes 同样提供了将宿主机的目录挂载到容器中的配置，就是 `hostPath`

```yaml {25-26}
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 2
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.8
          ports:
            - containerPort: 80
          volumnMounts:
            - mountPath: /usr/share/nginx/html
              name: nginx-vol
      volumns:
        - name: nginx-vol
          hostPath:
            path: "/opt/k8s-pratice/nginx-vol"
```
