# 牛刀小试:我的第一个容器化应用

Kubernates 建议我们将容器的定义、参数、配置等信息存储在一个 YAML 文件中，并通过 `kubectl create -f <fileName>`运行。

案例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 2
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.7.9
          ports:
            - containerPort: 80
```

像上面这样一个 YAML 文件在 K8S 中就是一个 API 对象，我们给该对象的各个字段填好值交给 K8S 之后，K8S 会创建响应的 API 资源

字段解释：

1. kind： 资源类型

问题：

1. Pod 创建后一直处于 Pending 状态

   环境信息：只有一个 Master 节点

   产生原因：K8S 默认不允许在 Master 节点部署容器

   解决方案：`kubectl taint nodes --all node-role.kubernetes.io/master-`
