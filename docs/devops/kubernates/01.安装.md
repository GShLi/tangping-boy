## 一、环境准备

### 1. 服务器环境

::: tip 提示
节点 CPU 核数必须是 >= 2 核 ，否则 k8s 无法启动 DNS

网络：最好设置为 本地网络连通的 DNS, 否则网络不通，无法下载一些镜像

linux 内核： linux 内核必须是 4 版本以上，因此必须把 linux 核心进行升级

准备 3 台虚拟机环境，或者是 3 台阿里云服务器都可。

k8s-master01: 此机器用来安装 k8s-master 的操作环境

k8s-node01: 此机器用来安装 k8s node 节点的环境

k8s-node02: 此机器用来安装 k8s node 节点的环境
:::

### 2. 依赖环境

1. 给每台机器设置主机名

    ```bash
    hostnamectl set-hostname fort001
    hostnamectl set-hostname fort002
    hostnamectl set-hostname fort003
    hostnamectl set-hostname fort004
    hostnamectl set-hostname fort005
    hostnamectl set-hostname fort006
    hostnamectl set-hostname fort007
    hostnamectl set-hostname fort008

    # 查看主机名
    hostname
    ```

2. 修改 /etc/hosts 文件，配置 IP host 映射关系

    ```bash
    vim /etc/hosts

    192.168.0.121  fort001
    192.168.0.122  fort002
    192.168.0.123  fort003
    192.168.0.124  fort004
    192.168.0.125  fort005
    192.168.0.126  fort006
    192.168.0.127  fort007
    192.168.0.128  fort008
    ```

3. 安装环境依赖

    ```bash
    yum install -y conntrack ntpdate ntp ipvsadm ipset jq iptables curl sysstat libseccomp wget vim net-tools git iproute lrzsz bash-completion tree bridge-utils unzip bind-utils gcc
    ```

4. 所有节点禁用 SELinux 和 Firewalld 服务：

    ```bash
    systemctl stop firewalld
    systemctl disable firewalld
    # 置空 iptables
    yum -y install iptables-services && systemctl start iptables && systemctl enable iptables && iptables -F && service iptables save

    # 重启后生效
    sed -i 's/enforcing/disabled/' /etc/selinux/config 
    ```

5. 所有节点禁用 swap 分区：

    ```bash
    # 临时禁用 swap 分区
    swapoff -a

    # 永久禁用 swap 分区
    vim /etc/fstab 
    # 注释掉下面的设置
    # /dev/mapper/centos-swap swap
    # 之后需要重启服务器生效
    ```

6. 所有节点添加网桥过滤和地址转发功能：

    ```bash
    cat > /etc/sysctl.d/kubernetes.conf << EOF
    net.bridge.bridge-nf-call-ip6tables = 1
    net.bridge.bridge-nf-call-iptables = 1
    net.ipv4.ip_forward = 1
    EOF

    # 然后执行，生效
    sysctl --system
    ```

7. 升级 Linux 内核为 4.44 版本

    ```bash
    rpm -Uvh http://www.elrepo.org/elrepo-release-7.0-4.el7.elrepo.noarch.rpm
    # 安装内核
    yum --enablerepo=elrepo-kernel install -y kernel-lt
    # 查看当前的所有内核版本
    cat /boot/grub2/grub.cfg | grep menuentry
    # 查看当前启动内核版本
    grub2-editenv list
    # 修改启动内核版本，设置开机从新内核启动
    grub2-set-default 'CentOS Linux (5.7.7-1.el7.elrepo.x86_64) 7 (Core)'
    # 注意：设置完内核后，需要重启服务器才会生效。
    # 查询内核
    uname -r
    ```

8. 调整内核参数，对于 k8s

    ```bash
    cat > kubernetes.conf << EOF
    net.bridge.bridge-nf-call-iptables=1
    net.bridge.bridge-nf-call-ip6tables=1
    net.ipv4.ip_forward=1
    net.ipv4.tcp_tw_recycle=0
    vm.swappiness=0
    vm.overcommit_memory=1
    vm.panic_on_oom=0
    fs.inotify.max_user_instances=8192
    fs.inotify.max_user_watches=1048576
    fs.file-max=52706963
    fs.nr_open=52706963
    net.ipv6.conf.all.disable_ipv6=1
    net.netfilter.nf_conntrack_max=2310720
    EOF

    # 将优化内核文件拷贝到 /etc/sysctl.d/ 文件夹下，这样优化文件开机的时候能够被调用
    cp kubernetes.conf /etc/sysctl.d/kubernetes.conf
    # 手动刷新，让优化文件立即生效
    sysctl -p /etc/sysctl.d/kubernetes.conf
    ```

9. 调整系统临时区 — 如果已经设置时区，可略过

    ```bash
    # 设置系统时区为中国 / 上海
    timedatectl set-timezone Asia/Shanghai
    # 将当前的 UTC 时间写入硬件时钟
    timedatectl set-local-rtc 0
    # 重启依赖于系统时间的服务
    systemctl restart rsyslog
    systemctl restart crond
    ```

10. 关闭系统不需要的服务

11. 设置日志保存方式

    1. 创建保存日志的目录

        ```bash
        mkdir -p /var/log/journal
        ```

    2. 创建配置文件存放目录

        ```bash
        mkdir -p /etc/systemd/journald.conf.d
        ```

    3. 创建配置文件

        ```bash
        cat > /etc/systemd/journald.conf.d/99-prophet.conf <<EOF
        [Journal]
        Storage=persistent
        Compress=yes
        SyncIntervalSec=5m
        RateLimitInterval=30s
        RateLimitBurst=1000
        SystemMaxUse=10G
        SystemMaxFileSize=200M
        MaxRetentionSec=2week
        ForwardToSyslog=no
        EOF
        ```

    4. 重启 systemd journald 的配置

        ```bash
        systemctl restart systemd-journald
        ```

12. 打开文件数调整 （可忽略，不执行）

13. kube-proxy 开启 ipvs 前置条件

### 3.Docker 部署

1. 安装 docker

    ```bash
    yum install -y yum-utils
    # 紧接着配置一个稳定（stable）的仓库、仓库配置会保存到 /etc/yum.repos.d/docker-ce.repo 文件中
    yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
    
    # 查看 docker 可安装版本
    yum list docker-ce --showduplicates | sort -r
    # 更新 Yum 安装的相关 Docke 软件包 & 安装 Docker CE
    yum install -y docker-ce-20.10.11 docker-ce-cli-20.10.11 containerd.io

    # 设置开机启动
    systemctl enable docker.service
    ```

2. 设置 daemon 文件，需要注意的是要配置 docker 的 cgroupdriver：

    ```bash
    mkdir -p /etc/docker
    cat > /etc/docker/daemon.json << EOF
    {
    "exec-opts": ["native.cgroupdriver=systemd"]
    }
    EOF
    ```

3. 重启 docker 服务

    ```bash
    systemctl daemon-reload && systemctl restart docker && systemctl enable docker
    ```

4. 确认 docker cgroup driver

    ```bash
    docker info | grep group
    # 记住这个
    # Cgroup Driver: systemd
    ```

### 4. 安装 kubeadm

1. 所有节点的 kubernetes 镜像切换成国内源：

    ```bash
    cat > /etc/yum.repos.d/kubernetes.repo << EOF
    [kubernetes]
    name=Kubernetes
    baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64
    enabled=1
    gpgcheck=0
    repo_gpgcheck=0
    gpgkey=https://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg https://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg
    EOF
    ```

2. 安装 kubeadm、kubelet、kubectl

    ```bash
    yum install -y kubelet-1.23.0 kubeadm-1.23.0 kubectl-1.23.0
    ```

3. 启动 kubelet

    ```bash
    # 设置 kubelet 开机启动
    systemctl enable kubelet && systemctl start kubelet
    ```

## 集群安装

首先要覆盖 kubeadm 的镜像地址，因为这个是外网的无法访问，需要替换成国内的镜像地址，使用此命令列出集群在配置过程中需要哪些镜像：

```bash
kubeadm config images list
```

更改为阿里云的镜像地址：

```bash
kubeadm config images list  --image-repository registry.aliyuncs.com/google_containers
```

然后将镜像手动拉取下来，这样在初始化的时候回更快一些（还有一个办法就是直接在 docker 上把镜像 pull 下来，docker 只要配置一下国内源即可快速的将镜像 pull 下来）：

```bash
kubeadm config images pull  --image-repository registry.aliyuncs.com/google_containers
```

```bash
kubeadm init \
  --apiserver-advertise-address=192.168.0.121 \
  --image-repository registry.aliyuncs.com/google_containers \
  --kubernetes-version v1.23.0 \
  --service-cidr=10.96.0.0/12 \
  --pod-network-cidr=10.244.0.0/16 \
  --ignore-preflight-errors=all

# –apiserver-advertise-address # 集群通告地址 (master 机器 IP，这里用的万兆网）
# –image-repository # 由于默认拉取镜像地址 k8s.gcr.io 国内无法访问，这里指定阿里云镜像仓库地址
# –kubernetes-version #K8s 版本，与上面安装的一致
# –service-cidr #集群内部虚拟网络，Pod 统一访问入口，可以不用更改，直接用上面的参数
# –pod-network-cidr #Pod 网络，与下面部署的 CNI 网络组件 yaml 中保持一致，可以不用更改，直接用上面的参数

kubeadm join 192.168.0.121:6443 --token xfueku.9koyx9uvt0amk70e \
        --discovery-token-ca-cert-hash sha256:078a3e4ca6a1d9441c3e3037da5e1251bfd2073181e96a061605b036982f93c0
```

## flannel 插件

::: tip 提示
部署 flannel 网络插件 — 只需要在主节点执行
:::

部署容器网络，CNI 网络插件 （在 Master 上执行，著名的有 flannel、calico、canal 和 kube-router 等，简单易用的实现是为 CoreOS 提供的 flannel 项目），这里使用 Flannel 实现。

下载 `kube-flannel.yml`：

```bash
wget https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml
```

打开文件找到 net-conf.json 位置，并配置 kubeadm init 时配置的网段

```yaml
net-conf.json: |
  {
    "Network": "10.244.0.0/16",
    "Backend"": {
      "Type": "vxlan"
  }
}
```

修改配置之后安装组件

```bash
kubectl apply -f kube-flannel.yml
```

查看 flannel pod 状态（必须要为 `Running` 状态，如果 `kube-flannel` 起不来，那么就用 `kubectl describe pod kube-flannel-ds-f5jn6 -n kube-flannel` 命令查看 pod 起不来的原因，然后去搜度娘获取解决方案）：

```bash
kubectl get pod --all-namespaces
```

## 问题

### kubelet 启动失败

kubelet cgroup driver: "cgroupfs" is different from docker cgroup driver: "systemd"

```bash
journalctl -u kubelet
# 错误
# kubelet cgroup driver: "cgroupfs" is different from docker cgroup driver: "systemd"
```

原因：
Kubelet 的 cgroup 驱动程序与 Docker 的 cgroup 驱动程序不一致
解决办法：

```bash
# 修改配置文件添加 Environment
vim /etc/systemd/system/kubelet.service.d/10-kubeadm.conf
# Environment="KUBELET_CGROUP_ARGS=--cgroup-driver=systemd"
# 如若 Environment 有多个配置可用逗号分隔
systemctl daemon-reload && systemctl restart kubelet
systemctl status kubelet
```

### kubelet 日志提示 node not found

原因：apiserver 没有启动成功

`docker ps -a` 查看 apiserver 对应的容器有没有起来

`docker logs <containerID>` 查看容器日志

日志内容如下：

```text
W0123 14:30:23.552243       1 clientconn.go:1251] grpc: addrConn.createTransport failed to connect to {127.0.0.1:2379 0  <nil>}. Err :connection error: desc = "transport: Error while dialing dial tcp 127.0.0.1:2379: connect: connection refused". Reconnecting...
```

最终解决方法

修改初始化时指定的 config 文件中的 advertise-address 相关的配置，本次安装时制定为宿主机的 IP 地址了

```text
kubeadm join 10.0.12.3:6443 --token abcdef.0123456789abcdef \
    --discovery-token-ca-cert-hash sha256:5ec40b014935cf24085308219b45aace0fb5f35846175d2c23a45aafdce101c3
```
