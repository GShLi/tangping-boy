# 五步配置 GitHub SSH

::: tip 从 GitHub 上 clone 项目有如下两种方式

**HTTPS**：

特点：简单易用、方便，适合初学者。通过 `git clone [HTTPS URL]` 可以直接将项目 clone 到本地。

缺点：每次 `fetch` 和 `push` 的时候都需要输入账号和密码进行验证。

**SSH URL**：

特点：在 clone 项目之前需要配置和添加好 SSH Key，当我们在 `fetch` 和 `push` 的时候不需要再次输入账号密码进行验证了。

缺点：配置流程较为复杂，对于初学者可能较为困难。
:::

## 一、查看本机是否创建了密钥

```bash
cd ~/.ssh
ls
```

执行完这两条命令后查看控制台是否出现了 **id_rsa.pub** 或 **id_dsa.pub** 这两个文件，如果有则可以跳过第二步。

## 二、生成密钥

`ssh-keygen -t rsa -C "your email"`

-t 指定密钥类型，默认是 rsa

-C 设置注释文件，可以填写你的 GitHub 注册邮箱

-f 指定密钥存放路径

因为我没有用 -f 指定密钥存放路径，因此在这需要手动输入存放路径。

不输入则使用默认路径（ 括号里的路径 ）

`Generating public/private rsa key pair.`
`Enter file in which to save the key (/Users/username/.ssh/id_rsa):`

如果该路径下已经有密钥文件则会询问你是否需要覆盖文件

```bash
/Users/username/.ssh/id_rsa already exists.
Overwrite (y/n)? y
```

接下来则是让我们输入密码，此密码并非 GitHub 密码，而是我们以后 push 时需要输入的密码，如果我们不设置则可以直接回车，这样我们以后 push 就不用输入密码了。

`Enter passphrase (empty for no passphrase):`
`Enter same passphrase again:`

上述步骤完成之后就会看到密钥生成成功的提示信息

```bash

Your public key has been saved in /Users/username/.ssh/id_rsa.pub.
```

## 三、添加 SSH Key 到 GitHub 上去

确保启动 SSH 代理

```bash
eval "$(ssh-agent -s)"
```

为 SSH Key 启动 SSH 代理

```bash
ssh-add ~/.ssh/id_rsa
```

之后将 id_rsa.pub 文件内的内容复制到粘贴板。

登录 GitHub，依次点击：右上角头像 -> Settings -> SSH and GPG keys -> New SSH Key

然后输入 刚刚复制的 id_rsa.pub 再起一个 Title 名称就完成了。

## 四、测试 SSH Key 是否配置成功

```bash
ssh -T git@github.com
```

当出现如下信息的时候就代表认证成功了。

```bash
Hi [Username]! You've successfully authenticated, but GitHub does not provide shell access.
```

## 五、实际 clone、commit、push 测试

我们复制一个自己 GitHub 上项目的 SSH 链接

然后通过如下命令 clone 项目到本地

```bash
git clone 'Your SSH URL'
```

之后我们随便修改一些文件之后 push 到 GitHub

此时我们是不需要输入账号和密码的（ 如果你在生成 SSH Key 的时候没有设置密码 ）
