# 线程的状态转换

## 线程有哪些状态？

线程的状态

1. New (新创建)
2. Runnable (可运行)
3. Blocked (被阻塞)
4. Waiting (等待)
5. Time-Waiting (计时等待)
6. Terminated (被终止)

线程在任意一个时间点只能处于一种状态。

其中 Blocked、Waiting、Time-Waiting 统称为阻塞状态。

### New 新创建

当 `new Thread ()` 一个线程但没有调用 `start ()` 方法时，线程处于 New 状态。

### Runnable 可运行

当调用 Thread 对象的 start 方法时，线程状态将从 New 转换为 Runnable。

Java 线程的 Runnable 状态对应操作系统中的 Ready 和 Running 状态。也就是说处于 Runnable 状态的 Java 线程此时可能正在执行也可能正准备执行。

### Blocked 被阻塞

只有线程进入 synchronized 代码块且没有拿到锁时，线程进入 Blocked 状态。

当线程获取到锁后，状态由 Blocked 转为 Runnable 状态。

### Waiting 等待

进入 Waiting 状态的几种情况：

1. 调用没有 timeout 参数的 `obj.wait ()` 方法
2. 调用没有 timeout 参数的 `thread.join ()` 方法
3. 调用 `LockSupport.park ()` 方法
