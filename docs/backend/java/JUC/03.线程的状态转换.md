# 线程的状态转换

## 线程有哪些状态？

线程的状态

1. New (新创建)
2. Runnable (可运行)
3. Blocked (被阻塞)
4. Waiting (等待)
5. Time-Waiting (计时等待)
6. Terminated (被终止)

线程在任意一个时间点只能处于一种状态。

其中 Blocked、Waiting、Time-Waiting 统称为阻塞状态。

### New 新创建

当 `new Thread ()` 一个线程但没有调用 `start ()` 方法时，线程处于 New 状态。

### Runnable 可运行

当调用 Thread 对象的 start 方法时，线程状态将从 New 转换为 Runnable。

Java 线程的 Runnable 状态对应操作系统中的 Ready 和 Running 状态。也就是说处于 Runnable 状态的 Java 线程此时可能正在执行也可能正准备执行。

### Blocked 被阻塞

只有线程进入 synchronized 代码块且没有拿到锁时，线程进入 Blocked 状态。

当线程获取到锁后，状态由 Blocked 转为 Runnable 状态。

### Waiting 等待

进入 Waiting 状态的几种情况：

1. 调用没有 timeout 参数的 `obj.wait ()` 方法
2. 调用没有 timeout 参数的 `thread.join ()` 方法
3. 调用 `LockSupport.park ()` 方法

### Time-Waiting 计时等待

## Object 对象中的基本方法

```java
public final void wait () throws InterruptedException

public final void wait (long timeout) throws InterruptedException

public final void wait (long timeout, int nanos) throws InterruptedException

public final void notify ();

public final void notifyAll ();
```

### wait ()

wait 方法分为带参数和不带参数两种。

在线程获取到锁之后，发现当前不满足线程执行的条件，可以调用锁对象的 wait 方法释放锁，并将该线程放到与锁关联的等待队列中。

此时线程从 `Runnable` 转换为 `Waiting` 状态。

带参数的 wait 方法与 不带参数的 wait 方法类似，在线程获取到锁之后，发现当前不满足线程执行的条件，可以调用锁对象的 wait 方法释放锁并指定一个 timeout 时间，并将该线程放到与锁关联的等待队列中，当线程在等待队列中等待的时间超过 timeout 设置的时间后会被自动从等待队列中移出。

### notify () 和 notifyAll ()

锁对象 notify 方法的作用是随机唤醒一个与当前锁关联的线程等待队列中的线程。(随机性取决于 JVM 的实现，JDK1.8 Hotspot 是顺序唤醒的)

notifyAll 方法则是唤醒当前锁所关联的线程等待队列中所有的线程。

所谓的唤醒就是让等待队列中的线程状态转换为 Runnable 活 Blocked 状态 (线程未获取到 synchronized 代码块锁)

notify 或 notifyAll 会立即释放锁吗？

```java
public void test () {
    Object objectLock = new Object ();
    synchronized (objectLock){
        objectLock.notifyAll ();
        while (true){
        }
    }
}
```

答案是否定的，例如上述代码，objectLock 锁在调用完 notifyAll 之后进入了一个死循环，这会导致 objectLock 锁所关联的线程等待队列中的所有队列永远不会运行，因为 objectLock 锁始终没有被释放。

```java
public static void main (String [] args) {
    Object objectLock = new Object ();
    new Thread (() -> {
        synchronized (objectLock) {
            System.out.println (Thread.currentThread ().getName () + "开始执行！");
            try {
                objectLock.wait ();
            } catch (InterruptedException e) {
                e.printStackTrace ();
            }
            System.out.println (Thread.currentThread ().getName () + "被唤醒！");
        }
    }, "t1").start ();

    new Thread (() -> {
        synchronized (objectLock) {
            objectLock.notify ();
            System.out.println (Thread.currentThread ().getName () + "发出通知！");
        }
    }, "t2").start ();
}
```
