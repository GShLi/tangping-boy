# 分类维护 API - 跨域问题的原理及处理方案

## 什么是跨域

[跨源资源共享（CORS）](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/CORS)

跨域：浏览器不能执行其他网站的脚本，是由于同源策略导致的，是浏览器对 JavaScript 施加的安全限制。

其他网站？

同源策略：如果两个 URL 的协议、端口和主机都相同的话，则这两个 URL 是同源的。

同源策略是一个重要的安全策略，它用于限制一个源的文档或者它加载的脚本如何能与另一个源的资源进行交互。

详情见: [浏览器的同源策略](https://developer.mozilla.org/zh-CN/docs/Web/Security/Same-origin_policy)

综上，跨域问的产生是因为浏览器在发送复杂 XHR 请求时会先发送预请求判断目标服务器是否开启了跨域资源共享，只有服务器向其他源开启共享后浏览器才会发送真正的资源请求。

因此解决的方案有两种：

- 资源服务器开启跨域
- 通过代理的方式将其他源的请求转发到资源服务器，实现同源资源共享
