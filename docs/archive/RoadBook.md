# 编程路书

> 这不是一份"要学什么"的清单——AI 可以在一秒内生成长达 100 页的技术目录。这份路书要回答的问题是：**在 AI 时代，我如何成为一个不可替代的程序员？**

---

## 核心哲学

**AI 能生成答案，但不能替你思考。**

这份路书的三个原则：

1. **写你踩过的坑、做过的决策、读过的书引发的思考**——这些 AI 无法替你生成
2. **从「知识搬运」转向「思考输出」**——概念笔记价值趋近于零，决策过程和失败复盘价值最高
3. **深度优先于广度**——20 篇高质量文章比 300 篇骨架有价值得多

---

## 内容层次金字塔

```
        ┌──────────────┐
       /  思考层         \
      /  决策、权衡、复盘   \   ← 最有价值，AI 不可替代
     /────────────────────\
    /   实践层               \
   /   项目实战、问题排查、     \  ← 有价值，展示解决能力
  /    性能调优                \
 /──────────────────────────────\
/    知识层                      \
/   概念笔记、API 速查、配置记录     \ ← 价值最低，AI 可直接生成
└──────────────────────────────────┘
```

**行动准则**：把 80% 的精力从知识层移到实践层和思考层。

## 第二阶段：走向架构师（进行中 →）

> 这不是学一门新框架，而是**思维方式从「怎么实现」到「为什么这样设计」的转变**。

### 2.1 核心能力培养

| 能力 | 如何锻炼 | 网站产出 |
|------|---------|---------|
| **系统设计** | 每个新需求先画架构图，写设计文档 | 架构决策记录 (ADR) |
| **技术选型** | 做选型时写对比文档（A vs B，各自适用场景） | 技术选型笔记 |
| **领域建模** | 学习 DDD，在 iPostYou / doublemall 项目中实践 | DDD 实践记录 |
| **分布式架构** | 从高并发笔记出发，结合实际项目理解 CAP、BASE | 分布式设计笔记 |
| **性能优化** | 拿一个真实接口做压测 → 优化 → 再压测 | 性能调优案例 |

### 2.2 学习路径（按需而非按目录）

```
实际问题驱动                    技术学习
─────────────                  ──────────
系统越来越慢          →         性能分析 + JVM 调优
单机扛不住了          →         分布式架构 + Redis 集群
代码越写越乱          →         设计模式 + DDD
系统间数据不一致       →         分布式事务 + 消息队列
需要做数据报表         →         数据仓库 + ETL
部署太麻烦            →         Docker + CI/CD
```

**原则**：不在没有实际问题时提前学习技术。空学 K8S 不如先手动部署一个服务，痛了再学。

### 2.3 架构设计书单（精读，不是速读）

| 书名 | 为什么读 | 网站产出 |
|------|---------|---------|
| 《设计模式》GoF | 理解设计语汇，能读懂框架源码 | 用 Java 重新实现一遍，代码上传 GitHub |
| 《领域驱动设计》Eric Evans | 学会和业务方对话，建模真实世界 | iPostYou 项目用 DDD 重新建模 |

| 《重构》 | 改善既有代码的嗅觉和方法 | 找一个老模块重构，记录前后对比 |

## 第四阶段：AI 时代的生存法则

### 不要做的事

- ❌ **不要和 AI 比写代码速度**——你比不过
- ❌ **不要花大量时间背 API 和语法**——AI 记得比你准
- ❌ **不要学「全栈」为学而学**——AI 让通才贬值，专才 + 系统思维升值
- ❌ **不要抄概念到笔记里**——那是 2023 年的事，现在 AI 一秒能生成一千条
- ❌ **不要同时学太多东西**——广度是 AI 的优势，深度才是人的优势

### 要做的事

- ✅ **培养 AI 无法替代的判断力**：什么场景用什么方案、为什么选 A 不选 B。把这个思考过程写下来。
- ✅ **学会用 AI 放大自己**：用 AI 辅助 code review、架构分析、文档编写、重构
- ✅ **提升沟通和表达能力**：能把复杂技术讲清楚的人永远是稀缺的（这个网站就是你的训练场）
- ✅ **积累领域知识**：电商（doublemall）、邮政（iPostYou）——领域知识 + 技术能力 = 不可替代
- ✅ **公开思考和输出**：一个活跃的技术博客，在招聘市场上比简历有说服力 10 倍

### 你的差异化方向

结合已有积累，你的护城河在：

```
Java 深度 + Spring 源码级理解 + 电商/邮政领域知识 + 架构设计能力
                              ↑
                         AI 最难替代的部分
```
